# Elliptic Curve Integrated Encryption Scheme (ECIES) with AES-GCM

This document summarizes the ECIES encryption process using elliptic curve cryptography with symmetric encryption (e.g., AES-GCM), and explains how standard curves like `secp256r1` are defined.

## Overview: ECIES in Practice

**ECIES** (Elliptic Curve Integrated Encryption Scheme) is a hybrid cryptographic scheme combining:

- **Asymmetric key exchange** (via elliptic curves)
- **Symmetric encryption** (e.g., AES-GCM) for message confidentiality and integrity

It is used in modern protocols and embedded systems due to its efficiency and strong security properties.

---

## Core Concepts

| Concept        | Description |
|----------------|-------------|
| Elliptic Curve | Defines a group over a finite field using the equation $y^2 = x^3 + ax + b \mod p$ |
| Base Point $G$ | A public point of prime order used to generate key pairs |
| Private Key $d$ | A random scalar in $[1, n-1]$ |
| Public Key $Q = dG$ | A point on the curve derived from the private key |
| Ephemeral Key $k$ | One-time-use private key used during encryption |
| Shared Secret $S = kQ_B$ | Used to derive a symmetric key |
| AES-GCM | Authenticated encryption algorithm ensuring confidentiality and integrity |

---

## ECIES Encryption Process

### **Bob** (receiver):

- Chooses private key $d_B$ and computes public key $Q_B = d_B G$

### **Alice** (sender):

1. Chooses random ephemeral key $k$
2. Computes ephemeral public key $R = kG$
3. Computes shared secret point $S = kQ_B = (x, y)$
4. Extracts $S_x$ (the x-coordinate)
5. Derives AES key: `K = SHA256(S_x)`
6. Encrypts message $m$ using `AES-GCM(K)`
7. Sends $(R, \text{ciphertext})$

### **Bob** (receiver):

1. Computes shared secret $S' = d_B R = (x, y)$
2. Extracts $S_x$ and derives `K = SHA256(S_x)`
3. Decrypts ciphertext using `AES-GCM(K)`

---

## Example (Toy Curve over $\mathbb{F}_{17}$)

- Curve: $y^2 = x^3 + 2x + 2 \mod 17$
- Base point: $G = (5, 1)$
- Bob's private key: $d_B = 7$
- Bob's public key: $Q_B = 7G = (0, 6)$

### Alice:

- Chooses ephemeral key $k = 3$
- Computes $R = 3G = (10, 6)$
- Computes shared secret $S = 3Q_B = (7, 6)$
- Derives AES key: $K = S_x = 7$
- Encrypts: $m = 13 \Rightarrow c = 13 + 7 = 20 \mod 17 = 3$

### Bob:

- Computes $S' = 7R = (7, 6)$
- Derives $K = 7$
- Decrypts: $3 - 7 = -4 \mod 17 = 13$

**Recovered message: 13**

## Why Use AES-GCM Even With Fresh Keys?

Even if each message uses a unique key, you still need **authenticated encryption**:

| Reason | Explanation |
|--------|-------------|
| Authentication | Prevents tampering and forgery |
| Replay protection | Each message uses a fresh key + IV |
| Confidentiality ≠ Integrity | AES-CTR, AES-CBC are not secure without a MAC |
| Secure-by-default | AES-GCM combines encryption + authentication safely |

---

## Restrictions on Keys

| Type         | Range                        | Requirement |
|--------------|------------------------------|-------------|
| Private key $d$ | $1 \leq d < n$               | Must be random and secret |
| Ephemeral key $k$ | $1 \leq k < n$               | Must be random, fresh, never reused |
| Order $n$    | Prime order of $G$           | Ensures strong discrete log security |

---

## 📘 What Is the Order of $G$?

- $G$ has order $n$ if $nG = \mathcal{O}$ (identity)
- Typically, $n$ is a large **prime** that divides the total number of points $\#E(\mathbb{F}_p)$
- Cryptographers **precompute** $G$ with known prime order and publish it in standards
